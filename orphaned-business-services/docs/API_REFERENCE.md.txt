# ClawPurse Gateway - Comprehensive API Reference

## Overview
ClawPurse Gateway provides a comprehensive set of RESTful and event-driven APIs for blockchain and financial services.

## Authentication

### User Registration
- **Endpoint**: `/auth/register`
- **Method**: POST
- **Request Body**:
```json
  {
    "email": "user@example.com",
    "password": "secure_password",
    "firstName": "John",
    "lastName": "Doe"
  }
```
- **Response**: 
  - 201: User created successfully
  - 400: Registration failed

### User Login
- **Endpoint**: `/auth/login`
- **Method**: POST
- **Request Body**:
```json
  {
    "email": "user@example.com",
    "password": "secure_password"
  }
```
- **Response**:
  - 200: Login successful (returns JWT token)
  - 401: Invalid credentials

## Wallet Management

### Create Wallet
- **Endpoint**: `/wallets`
- **Method**: POST
- **Authentication**: Required
- **Request Body**:
```json
  {
    "type": "blockchain",
    "currency": "BTC"
  }
```
- **Response**: Created wallet details

### List Wallets
- **Endpoint**: `/wallets`
- **Method**: GET
- **Authentication**: Required
- **Response**: Array of user wallets

## Transaction Management

### Create Transaction
- **Endpoint**: `/transactions`
- **Method**: POST
- **Authentication**: Required
- **Request Body**:
```json
  {
    "fromWalletId": "wallet_123",
    "toWalletId": "wallet_456",
    "amount": 0.01,
    "currency": "BTC"
  }
```
- **Response**: Transaction details

### List Transactions
- **Endpoint**: `/transactions`
- **Method**: GET
- **Authentication**: Required
- **Query Parameters**:
  - `walletId`: Filter by specific wallet
  - `type`: Filter by transaction type
- **Response**: Array of transactions

## Error Handling

### Common Error Codes
- 400: Bad Request
- 401: Unauthorized
- 403: Forbidden
- 404: Not Found
- 500: Internal Server Error

## Rate Limiting
- Maximum 100 requests per minute
- Exceeded limit returns 429 Too Many Requests

## Security
- All communications use HTTPS
- JWT tokens expire every 30 minutes
- Multi-factor authentication available

## Webhook Events

### Blockchain Events
- `transaction.created`
- `transaction.confirmed`
- `wallet.created`

### System Events
- `user.registered`
- `service.status`

## Versioning
- Current API Version: v1
- Deprecated versions will be supported for 6 months after deprecation notice

## Conclusion
Comprehensive, secure, and flexible API for blockchain and financial services.
