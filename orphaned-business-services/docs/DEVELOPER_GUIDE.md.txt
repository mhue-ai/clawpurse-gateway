## Getting Started

### Prerequisites
- Node.js 20+
- Docker
- Kubernetes Cluster
- Git
- TypeScript

### Installation
```bash
# Clone the repository
git clone https://github.com/mhue-ai/clawpurse-gateway.git
cd clawpurse-gateway

# Install dependencies
npm install

# Setup development environment
npm run dev:setup
```

## Development Environment

### Local Setup
- Use Docker for consistent development environment
- Utilize Minikube or Kind for local Kubernetes testing
- Recommended IDE: Visual Studio Code

### Environment Variables
Create a `.env` file with necessary configurations:
```
NODE_ENV=development
DATABASE_URL=postgresql://localhost/clawpurse
JWT_SECRET=your_secret
```

## Contribution Guidelines

### Branching Strategy
- `main`: Stable production code
- `develop`: Integration branch
- `feature/`: New feature development
- `hotfix/`: Critical bug fixes

### Pull Request Process
1. Fork the repository
2. Create a feature branch
3. Commit with descriptive messages
4. Run all tests
5. Submit pull request to `develop` branch

### Coding Standards
- Follow TypeScript best practices
- Use ESLint for code linting
- Maintain 80%+ test coverage
- Write comprehensive documentation

## Development Workflow

### Daily Development
```bash
# Start development server
npm run dev

# Run tests
npm test

# Lint code
npm run lint

# Build project
npm run build
```

### Creating a New Service
1. Generate service scaffold
```bash
npm run generate:service BlockchainService
```

2. Implement service logic
3. Write comprehensive tests
4. Update documentation

## Testing

### Test Types
- Unit Tests
- Integration Tests
- End-to-End Tests

### Running Tests
```bash
# Run all tests
npm test

# Run specific test suite
npm run test:unit
npm run test:integration
```

## Deployment

### Local Deployment
```bash
# Build Docker image
docker build -t clawpurse-gateway .

# Run local Kubernetes
kubectl apply -f k8s/
```

### CI/CD
- Automated testing on pull requests
- Automatic deployment to staging
- Manual approval for production
